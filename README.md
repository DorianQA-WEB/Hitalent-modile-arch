# Hitalent Module — Управление подразделениями и сотрудниками

REST API для управления иерархической структурой подразделений и сотрудниками. Проект реализован на **FastAPI** с асинхронным подключением к **PostgreSQL** через **SQLAlchemy**, контейнеризацией через **Docker** и фоновыми задачами через **Celery + Redis**.

## Функциональность

- **CRUD для подразделений** — создание, чтение, обновление, удаление с поддержкой иерархии (родитель-потомок)
- **Древовидная структура** — получение подразделения со всеми вложенными дочерними отделами до заданной глубины (до 5 уровней)
- **Управление сотрудниками** — создание сотрудников в рамках подразделения
- **Два режима удаления подразделения**:
  - `cascade` — каскадное удаление со всеми сотрудниками и дочерними отделами
  - `reassign` — перевод сотрудников в другой отдел перед удалением
- **Фоновые задачи** — через Celery + Redis
- **Логирование** — через Loguru с уникальным ID запроса
- **CORS** — разрешены все источники
- **Docker-контейнеризация** — готовый `docker-compose.yml` для быстрого развёртывания

## Быстрый старт

### Запуск через Docker Compose

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-username/Hitalent-module222.git
   cd Hitalent-module
   ```

2. **Создайте файл `.env`** в корне проекта (см. [Переменные окружения](#переменные-окружения)).

3. **Запустите сервисы:**
   ```bash
   docker-compose up --build
   ```

4. **Примените миграции:**
   ```bash
   docker-compose exec web alembic upgrade head
   ```

5. **Откройте документацию:** [http://localhost:8000/docs](http://localhost:8000/docs)

### Локальный запуск (без Docker)

1. **Убедитесь, что PostgreSQL и Redis запущены.**

2. **Создайте виртуальное окружение и установите зависимости:**
   ```bash
   python -m venv venv
   source venv/bin/activate   # Linux/Mac
   venv\Scripts\activate      # Windows
   pip install -r requirements.txt
   ```

3. **Настройте `.env` файл** с параметрами подключения к БД и Redis.

4. **Примените миграции:**
   ```bash
   alembic upgrade head
   ```

5. **Запустите приложение:**
   ```bash
   uvicorn app.main:app --reload
   ```

6. **Запустите Celery worker** (в отдельном терминале):
   ```bash
   celery -A app.main.celery worker --loglevel=info
   ```

## API Endpoints

### Подразделения

| Метод    | Endpoint                        | Описание                                      |
|----------|---------------------------------|-----------------------------------------------|
| `POST`   | `/departments/`                 | Создать новое подразделение                   |
| `GET`    | `/departments/{id}`             | Получить подразделение с древовидной структурой |
| `PATCH`  | `/departments/{id}`             | Обновить подразделение (имя или родителя)     |
| `DELETE` | `/departments/{id}?mode=cascade`| Удалить подразделение (каскадно)              |
| `DELETE` | `/departments/{id}?mode=reassign&reassign_to_department_id={id}` | Удалить с переводом сотрудников |

**Параметры `GET /departments/{id}`:**
- `depth` (int, 1–5) — глубина вложенности дочерних подразделений (по умолчанию 1)
- `include_employees` (bool) — включать ли сотрудников в ответ (по умолчанию `true`)

### Сотрудники

| Метод    | Endpoint                        | Описание                    |
|----------|---------------------------------|-----------------------------|
| `POST`   | `/departments/{id}/employees`   | Создать сотрудника в отделе |

## Модели данных

### Department (Подразделение)

| Поле        | Тип         | Описание                              |
|-------------|-------------|---------------------------------------|
| `id`        | Integer     | Первичный ключ                        |
| `name`      | String(200) | Название подразделения                |
| `parent_id` | Integer?    | ID родительского подразделения (FK)   |
| `created_at`| DateTime    | Дата создания                         |

- Поддерживает рекурсивную иерархию (родитель → потомки)
- Имена должны быть уникальны в пределах одного родителя

### Employee (Сотрудник)

| Поле           | Тип          | Описание                     |
|----------------|--------------|------------------------------|
| `id`           | Integer      | Первичный ключ               |
| `department_id`| Integer      | ID подразделения (FK)        |
| `full_name`    | String(255)  | ФИО сотрудника               |
| `position`     | String(255)  | Должность                    |
| `hired_at`     | DateTime?    | Дата приёма на работу        |
| `created_at`   | DateTime     | Дата создания записи         |

## Миграции

Управление схемой БД осуществляется через **Alembic**.

- **Создать новую миграцию:**
  ```bash
  alembic revision --autogenerate -m "описание изменений"
  ```

- **Применить миграции:**
  ```bash
  alembic upgrade head
  ```

- **Откатить миграцию:**
  ```bash
  alembic downgrade -1
  ```

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
DATABASE_URL=postgresql+asyncpg://test_user:testpass@db:5432/test_db
```

| Переменная     | Описание                              | Значение по умолчанию                          |
|----------------|---------------------------------------|------------------------------------------------|
| `DATABASE_URL` | URL для подключения к PostgreSQL      | `postgresql+asyncpg://test_user:testpass@db:5432/test_db` |

> **Примечание:** При локальном запуске замените `db` на `localhost` (или адрес вашего PostgreSQL-сервера).
